# 2026-05-10 Phase 7A 测试日志

## 目标

实现 opt-in SQLite 状态持久化，为后续真实邮箱 read-only 接入前补齐本地状态恢复能力。

## 当前约束

- 默认不启用 SQLite，保持内存模式和测试速度。
- 通过 `WISPERA_STATE_DB` 显式启用 SQLite。
- 不保存 API key、真实邮箱正文或完整 trace。
- 不持久化 pending approval，避免服务重启后误执行旧的危险动作。
- 不持久化 chat history，避免扩大隐私面。

## 实现内容

- 新增 `server/app/sqlite_state.py`。
- `MemoryStore` 支持可插拔 `StateStore`。
- `AgentRuntime.create()` 根据 `WISPERA_STATE_DB` 装配 SQLite 状态后端。
- 持久化 email preferences、reported email ids、notifications、scan history。
- 新增 `AgentRuntime.close()` / `MemoryStore.close()`，便于释放 SQLite 连接。
- 单元测试覆盖 MemoryStore 跨实例持久化和 AgentRuntime 工厂环境变量装配。

## 结论

Phase 7A 的最小目标已完成。项目现在可以在保持默认内存模式的同时，通过 `WISPERA_STATE_DB` 启用本地 SQLite 状态恢复。

面试时可以强调：

> 我没有把所有状态都持久化，而是只持久化邮件分拣长期需要的本地状态：偏好、通知、去重和扫描历史。pending approval 和 chat history 暂时不入库，这是基于安全和隐私边界的选择。
