## 目标

实现轻量 evaluation report 导出，把现有 mock evaluation 结果沉淀成可面试展示、可后续回归对比的 Markdown/JSON 报告。

## 当前约束

- 默认使用 rule classifier，离线运行。
- 不连接真实邮箱。
- 不修改邮箱状态。
- 不保存 API key、真实邮箱内容或完整长 rows。
- LLM report 只作为可选项，不进入单元测试。

- 新增 `server/app/email_eval_report.py`。

- 新增 tool：`email_eval_report`。
- `email_eval_report` 的 `output_path` 限制在 workspace 内。
- 单元测试覆盖 report tool 的 Markdown 写入。
